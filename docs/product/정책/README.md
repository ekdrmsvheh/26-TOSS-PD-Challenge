# 정책 문서 관리

> 이 폴더는 `TOSS 프디 챌린지 2026` 리서치 폴더의 최종 확정본(버전 표시 제거)을 스타터킷으로 옮긴 스냅샷이다.
> 버전 히스토리와 원본은 원본 리서치 폴더에 그대로 남아있다.  
> 최신 정책: [제품 데이터 정책](./제품%20데이터%20정책.md)

## 문서 역할

정책 문서는 UX flow와 프로토타입 사이에서 반복 사용되는 데이터 기준을 관리한다.

- 어떤 데이터를 입력받는가
- 어떤 상태로 해석하는가
- 어떤 기준으로 추천/정렬하는가
- 어떤 정보는 누구에게 숨기는가
- 어떤 예외를 어떻게 처리하는가
- AI가 할 수 있는 것과 하면 안 되는 것은 무엇인가

## 관리 원칙

- 화면 설명은 UX flow에, 데이터 판단 기준은 정책 문서에 둔다.
- 프로토타입 구현 중 상태값이 바뀌면 정책 문서를 먼저 갱신한다.
- 검증되지 않은 값은 확정처럼 쓰지 않고 `검증 필요`로 남긴다.
