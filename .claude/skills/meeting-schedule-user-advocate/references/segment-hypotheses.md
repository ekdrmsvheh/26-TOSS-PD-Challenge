# 목적 기반 세그먼트 가설

출처는 프로젝트의 `사용자 세그먼트/목적 기반 세그먼트_주최자.md`와 `목적 기반 세그먼트_참석자.md`다. GPT·Gemini·Claude 결과를 통합한 AI 산출물이므로 모든 항목을 `세그먼트 가설`로 취급한다. 실제 조사 응답자를 임의로 분류하지 않는다.

이 문서는 원문을 대체하는 압축본이 아니다. 관련 역할과 세그먼트를 고르는 라우팅 맵으로만 사용한다. 실제 판정에서는 SKILL.md의 시작 절차에 따라 해당 원문 전체를 읽고 일곱 항목을 모두 대조한다.

## 세그먼트 선택 원칙

- 사람을 고정 유형으로 분류하지 않는다. 같은 사람도 회의의 긴급도, 권한, 참석 역할, 기존 일정의 고정성에 따라 다른 목적을 가질 수 있다.
- 현재 장면에서 가장 강한 목적과 제약을 기준으로 선택한다.
- 사용자 정보가 부족하면 서로 다른 가설 2개를 나란히 검토한다.
- 기능 요구 목록보다 최종 목적, 두려움, 이탈 조건을 우선한다.

## 주최자·조정자

### H1. 의사결정 권한을 가진 효율 지향적 주최자

- 최종 목적: 필수 참석자를 확보하면서 조정 비용이 낮은 시간을 빠르게 확정해 프로젝트 정체를 막는다.
- 하위 목적: 응답을 신속히 모으기, 후보 차이 비교하기, 추천 근거 확보하기, 재조율 줄이기.
- 핵심 두려움: 미응답으로 인한 지연, 필수 참석자의 돌발 불참, 모호한 상태 때문에 다시 확인하는 것.
- 이탈 조건: 제품을 써도 개별 재촉·재확인·후보 재생성이 줄지 않거나 추천 근거를 믿을 수 없음.
- 검토 질문: 이 설계는 확정에 필요한 불확실성을 실제로 줄이는가, 아니면 보기 좋은 정보만 더하는가?

### H2. 의사결정 권한을 가진 관계 지향적 주최자

- 최종 목적: 참석자의 조건과 희생을 공정하게 반영해 팀의 신뢰와 협업 동력을 지키며 확정한다.
- 하위 목적: 비선호를 안전하게 수집하기, 특정인의 반복 희생 피하기, 불가피한 선택을 존중하는 언어로 설명하기.
- 핵심 두려움: 강제 배정과 기계적 추천으로 관계가 훼손되고 특정인의 불만이 누적되는 것.
- 이탈 조건: 자동화가 정성적 판단을 막거나 과도한 선호 입력과 무감각한 리마인드로 관계 비용을 키움.
- 검토 질문: 객관성을 내세운 설계가 사람의 사정과 주최자의 책임을 지워버리지 않는가?

### H3. 최종 권한이 없는 조정자

- 최종 목적: 참석자의 제약을 정확히 수집·정리해 권한자가 빠르게 결정하도록 돕고, 결정 과정의 불필요한 책임과 비난을 줄인다.
- 하위 목적: 응답 취합하기, 권한자의 기준으로 후보를 요약하기, 결정 후 빠르게 공지하기.
- 핵심 두려움: 불완전한 정보로 질책받기, 참석자와 권한자 사이에서 책임을 떠안기, 보고를 위해 다시 가공하기.
- 이탈 조건: 수집 결과를 다시 정리해야 하거나 권한자와 참석자에게 중복 설명해야 함.
- 검토 질문: 수집자와 최종 결정자가 다를 때 권한·상태·책임이 명확한가?

## 참석자

### A1. 개방적 조정형 참석자

- 최종 목적: 회의의 필요성과 자신의 역할을 이해한 뒤 가능한 범위에서 유연하게 협조하고 빠르게 확정된 시간을 안다.
- 하위 목적: 회의 맥락 이해하기, 간단히 가능 범위 전달하기, 변경 결과 확인하기.
- 핵심 두려움: 협조 의사가 무제한 양보로 해석되고, 반복 입력·확인이 발생하는 것.
- 이탈 조건: 간단한 응답보다 제품 사용이 번거롭거나 자신의 협조가 당연한 의무로 취급됨.
- 검토 질문: 협조적 사용자의 시간을 아낀다는 이유로 동의와 선호를 생략하지 않는가?

### A2. 선택적 제약형 참석자

- 최종 목적: 고정 불가, 조정 가능하지만 부담되는 시간, 편한 시간을 구분해 전달하고 그 경계가 존중된 채 한 번에 확정한다.
- 하위 목적: 조정 비용 표현하기, 후보별 자신의 부담 이해하기, 입력이 결정에 반영됐는지 확인하기.
- 핵심 두려움: `조정 가능`이 언제든 가능하거나 이미 변경에 동의한 것으로 오해되는 것.
- 이탈 조건: 불가·비선호·조정 가능의 차이가 사라지거나 입력 후에도 메신저 재확인이 반복됨.
- 검토 질문: 사용자의 회색지대를 충분히 표현하면서도 입력 부담을 과도하게 늘리지 않는가?

### A3. 엄격한 제약형 참석자

- 최종 목적: 반복되는 절대 불가 조건이 일관되게 반영되고, 충돌 없는 시간으로 빠르게 확정되어 같은 설명을 반복하지 않는다.
- 하위 목적: 절대 불가 전달하기, 변경 시 충돌 확인하기, 불가능하면 대안을 논의하기.
- 핵심 두려움: 명시한 제약이 무시되거나 불참 책임을 전가받고, 확정 후 변경으로 충돌이 생기는 것.
- 이탈 조건: 불가 시간이 계속 후보에 포함되거나 반복 수동 입력과 설명을 요구함.
- 검토 질문: 이 설계는 불가를 협상 가능한 선호로 취급하거나 예외 상황에서 조용히 무시하지 않는가?

## 공통 목적 충돌

| 충돌 | 확인할 결정 |
|---|---|
| 빠른 확정 vs 충분한 응답 | 어떤 역할의 미응답에서 추천·확정을 보류하는가 |
| 필수 참석자 보호 vs 선택 참석자 선호 | 어떤 조건은 후보를 막고 어떤 조건은 순위에만 반영하는가 |
| 관계적 배려 vs 입력 최소화 | 어떤 사유가 실제 결정에 필요하며 무엇은 묻지 않는가 |
| 객관적 추천 vs 사람의 책임 | 제품과 주최자 중 누가 추천·확정·설명을 맡는가 |
| 반복 조건 자동화 vs 회의별 맥락 | 저장된 조건을 언제 재확인하고 수정하게 하는가 |
| 투명한 근거 vs 개인정보 | 누구에게 어떤 사유와 조정 비용을 어느 수준까지 보여주는가 |
